About

dwbrite.com is an adventure in css and web servers. That is to say, this website is written in Go using gorilla for routing. A crude form of caching is implemented for executed templates - and eventually I'll add a filewatcher to cache everything else proper.

dwbrite.com used to be a Tumblr page where I wrote about my projects. I had taken a bit too much inspiration from a certain motherloving website. I wanted a bit more color, but I quite liked the layout as it was. The biggest change in the new version is the background. Blue, orange, and purple radial gradients merge into neutral tones to make reading easy on the eyes. And it's really pretty. Especially on tall pages.

Design

I designed this website with two things in mind:

  1. simple design language
  2. user friendliness

Simple design language means that interactive elements on the page should do what you expect, Things like self-descriptive buttons, and links being blue - including in navigation.

User friendliness means not being actively hostile to the user. Pages should load quickly without unnecessary bloat. Media that isn't strictly required should be opt-in or lazy-loaded. Contents of a page should be visible without the need for javascript. Links should be links and not lead you to garbage when opened in a new tab. Things like that.

Under the hood

The repo is split into source code and web content. Web content is maintained in the `root` folder. From there, each subdirectory can contain a `metadata.json` file which defines how the server should render its contents. A directory could be rendered as raw resources with a blacklist or whitelist, as a template executed with the contents of `index.html`, or as one of any other pre-made definitions. This allows for all kinds of fun tricks and easter eggs I may want to add in the future ;)
